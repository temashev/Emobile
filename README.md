# User Management API
Тестовое задание по разработке системы управления пользователями с использованием Django Rest Framework и PostgreSQL.
___

## Запуск проекта:
1. Клонирование репозитория:
```bash
git clone https://github.com/temashev/Emobile
cd Emobile
```
2. Настройка окружения:
- Необходимо создать `.env` файл на основе `.env.example`.
- Заполнить данные для подключения к PostgreSQL.
3. Установка зависимостей и активация виртуального окружения:
```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```
4. Применить миграции:
```bash
python manage.py migrate
```
5. Создание суперпользователя (Админа):
```bash
python manage.py createsuperuser
```
6. Запуск сервера:
``` bash
python manage.py runserver
```
Проект будет доступен по адресу: `http://127.0.0.1:8000/api/users/`
___
## Реализованный функционал

### Аутентификация и профиль
- **Регистрация**: Создание нового пользователя с автоматическим хешированием пароля.
- **Login/Logout**: Система блокирует вход для удаленных пользователей.
- **Управление профилем**: 
  - Просмотр и редактирование (ФИО, пароль) своего профиля через `/profile/<id>/`.
  - "Мягкое удаление": При удалении аккаунта устанавливается флаг `is_active=False`, данные сохраняются в БД, но доступ блокируется.

### Система прав доступа
Реализована схема управления ограничениями на базе БД:
- **Роли**: Доступ к ресурсам (`roles`, `permissions`, `profile`) регулируется через таблицу связей в БД.
- **Маппинг прав**: HTTP-методы автоматически соотносятся с типами разрешений:
  - `GET` : `view`
  - `POST` : `create`
  - `PUT/PATCH` : `update`
  - `DELETE` : `delete`
- **Иерархия**: 
  - **Админ**: Обладает полным доступом ко всем ресурсам и может физически удалять записи из БД.
  - **Пользователь**: Обычные пользователи могут редактировать только свои данные (проверка `IsOwnerOrReadOnly`).
- **Управление правами (для Админа)**: Реализован полный CRUD (создание, чтение, обновление, удаление) для ролей и прав доступа через API эндпоинты `/api/users/roles/` и `/api/users/permissions/`
